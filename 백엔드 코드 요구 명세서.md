# Missing Pet Finder — 백엔드 아키텍처 분석

> 제공된 아키텍처 다이어그램 및 백엔드 요구사항 명세서(`backend_requirement_doc.md`)를 기반으로 작성된 기술 분석 문서입니다.

## 1. 전체 아키텍처 개요

```
모바일 기기
    │  사진 전송 / 알람 전송
    ▼
┌─────────────┐
│   게이트웨이   │   ← API Gateway (인증, 라우팅, 요청 검증)
└──────┬──────┘
       │
┌──────▼──────┐        ┌──────────────────────┐
│  서비스 레이어 │ ◄────► │  관계형 DB (PostgreSQL) │
└──────┬──────┘        └──────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│              AI 파이프라인                 │
│                                         │
│  1. 품종 분류 (The Cat API)               │
│         ↓                               │
│  2. 특징 문장 증강 (Claude Sonnet)         │
│         ↓                               │
│  3. 임베딩 (Embedding Model)             │
│         ↓                               │
│  벡터 데이터베이스 (pgvector)               │
└─────────────────────────────────────────┘
```

### 주요 구성 요소 요약

| 컴포넌트 | 역할 | 기술 스택 |
|---|---|---|
| 게이트웨이 | 요청 라우팅, 인증, 알림 발송 | API Gateway (e.g., Kong, Nginx) |
| 서비스 레이어 | 비즈니스 로직 처리 | Node.js / Python (추정) |
| 관계형 DB | 사용자/게시글/채팅 데이터 저장 | PostgreSQL |
| AI 파이프라인 | 제보 사진 유사도 분석 | 3단계 파이프라인 (하단 상세) |
| 벡터 DB | 임베딩 벡터 저장 및 유사도 검색 | pgvector (PostgreSQL 확장) |

## 2. AI 파이프라인 상세 분석 — Two-Stage Filtering

제보 기능(`POST /api/tips/analyze`)의 핵심인 AI 파이프라인은 **두 단계의 필터링**을 통해 유사 반려동물을 탐색합니다.

```
제보 사진 입력 (3~5장)
        │
        ▼
┌───────────────────┐
│  Stage 1: 1차 필터  │  품종 분류 (The Cat API)
│  품종 레이블 추출    │
└────────┬──────────┘
         │  품종명 (e.g., "Pomeranian")
         ▼
┌───────────────────────────────┐
│  Stage 2: 2차 필터 (특징 문장   │  Claude Sonnet (LLM)
│  증강 + 임베딩 유사도 검색)      │
│                               │
│  2-1. 특징 문장 증강            │
│  2-2. 텍스트 임베딩             │
│  2-3. pgvector 유사도 검색     │
└───────────────────────────────┘
         │
         ▼
유사 실종견 상위 3건 + 유사도 점수 반환
```

---

### 2.1 Stage 1 — 품종 분류 (The Cat API)

**목적:** 입력 사진에서 동물의 품종(Breed)을 식별하여 **후보군을 1차로 좁히는** 필터 역할

| 항목 | 내용 |
|---|---|
| 외부 API | The Cat API (품종 분류 엔드포인트) |
| 입력 | 제보 사진 1장 이상 (JPEG/PNG) |
| 출력 | 품종 레이블 (e.g., `"Pomeranian"`, `"Shih Tzu"`) + 신뢰도 점수 |
| 후속 처리 | 도출된 품종으로 PostgreSQL의 `missing_pets` 테이블에서 `breed` 일치 건만 필터링 |

**필터링 효과:**
- 전체 실종견 DB → 동일 품종 후보군으로 범위 축소
- 이후 벡터 유사도 검색 비용 절감 (불필요한 후보 제거)

**고려사항:**
- 품종 분류 신뢰도가 낮을 경우 (`< threshold`) 필터를 스킵하고 전체 후보군으로 진행하는 fallback 로직 필요
- The Cat API의 품종 레이블과 DB의 `breed` 필드 값 간 정규화(매핑 테이블) 필요

---

### 2.2 Stage 2-1 — 특징 문장 증강 (Claude Sonnet)

**목적:** 이미지만으로는 포착하기 어려운 **세밀한 시각적 특징을 자연어 문장으로 변환**하여 검색 정밀도 향상

**입력 → 출력 예시:**

```
입력: 제보 사진 3~5장

출력 (증강된 특징 문장 예시):
"갈색 포메라니안, 귀 끝부분이 살짝 접혀 있으며 왼쪽 눈 아래 흰색 반점이 있음.
 털 길이는 중간 정도이며 꼬리가 등 위로 말려 있는 전형적인 포메라니안 체형.
 목 부위에 오렌지색 혼합 털이 관찰됨."
```

**Claude Sonnet 프롬프트 설계 방향:**

```
시스템: 당신은 반려동물 특징 분석 전문가입니다.
        제공된 이미지에서 다음 항목을 추출하여 한국어 문장으로 작성하세요:
        - 전체 털 색상 및 패턴
        - 얼굴 특징 (눈 색, 코 형태, 귀 형태)
        - 체형 및 크기 특징
        - 식별 가능한 특이 마킹 또는 신체적 특징
        - 꼬리, 발, 다리 특징

사용자: [이미지 N장 첨부]
```

**필요한 Anthropic API 파라미터:**
```json
{
  "model": "claude-sonnet-4-20250514",
  "max_tokens": 500,
  "messages": [
    {
      "role": "user",
      "content": [
        { "type": "image", "source": { "type": "base64", "media_type": "image/jpeg", "data": "..." } },
        { "type": "text", "text": "위 이미지의 반려동물 특징을 상세히 설명해주세요." }
      ]
    }
  ]
}
```

---

### 2.3 Stage 2-2 — 임베딩 (Embedding)

**목적:** Claude Sonnet이 생성한 **특징 문장을 고차원 벡터로 변환**하여 수치 유사도 계산에 사용

| 항목 | 내용 |
|---|---|
| 입력 | 특징 증강 문장 (한국어 자연어) |
| 출력 | 고차원 float 벡터 (e.g., 1536-dim) |
| 임베딩 모델 후보 | OpenAI `text-embedding-3-small`, Cohere `embed-multilingual`, 또는 오픈소스 `KLUE-BERT` |
| 다국어 지원 | 한국어 콘텐츠 → 다국어 지원 임베딩 모델 필수 |

**DB 등록 시 임베딩 생성 흐름 (사전 처리):**
```
실종 신고 등록 (POST /api/pets)
        │
        ▼
  사진 → Claude Sonnet → 특징 문장 생성
        │
        ▼
  특징 문장 → Embedding Model → 벡터
        │
        ▼
  pgvector에 벡터 저장 (pet_id 연결)
```

---

### 2.4 Stage 2-3 — pgvector 유사도 검색

**목적:** 제보 사진의 임베딩 벡터와 기존 DB의 벡터 간 **코사인 유사도를 계산**하여 상위 N건 반환

**pgvector 쿼리 예시:**
```sql
-- Stage 1 품종 필터 + Stage 2 벡터 유사도 검색 통합 쿼리
SELECT
    p.id,
    p.name,
    p.breed,
    p.location,
    p.photo,
    1 - (pe.embedding <=> $1::vector) AS similarity_score
FROM missing_pets p
JOIN pet_embeddings pe ON p.id = pe.pet_id
WHERE
    p.status = '실종'
    AND p.breed = $2              -- Stage 1: 품종 필터 적용
    AND ST_DWithin(               -- 위치 기반 추가 필터 (옵션)
        p.location_point,
        ST_MakePoint($3, $4),
        10000                     -- 반경 10km 이내
    )
ORDER BY similarity_score DESC
LIMIT 3;                          -- 상위 3건 반환
```

**유사도 점수 스케일링:**
```
코사인 유사도 (0~1) → 0~100 정수 변환
예: 0.94 → 94점, 0.78 → 78점, 0.61 → 61점
```

## 4. 데이터베이스 스키마 설계

### 4.1 PostgreSQL — 핵심 테이블

```sql
-- 실종 반려동물
CREATE TABLE missing_pets (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(50) NOT NULL,
    breed           VARCHAR(100),              -- Stage 1 필터링 키
    age             VARCHAR(20),
    gender          CHAR(1),                   -- 'M' | 'F'
    color           VARCHAR(100),
    location        TEXT NOT NULL,
    district        VARCHAR(50) NOT NULL,      -- 구 단위 필터링
    location_point  GEOGRAPHY(POINT, 4326),    -- 위경도 (PostGIS)
    last_seen       DATE NOT NULL,
    reward          INTEGER DEFAULT 0,
    photo           TEXT,
    photos          TEXT[],
    description     TEXT,
    status          VARCHAR(10) DEFAULT '실종', -- '실종' | '찾음'
    reporter_id     UUID REFERENCES users(id),
    views           INTEGER DEFAULT 0,
    likes_count     INTEGER DEFAULT 0,
    comments_count  INTEGER DEFAULT 0,
    created_at      TIMESTAMPTZ DEFAULT now(),
    updated_at      TIMESTAMPTZ DEFAULT now()
);

-- 지오공간 인덱스 (반경 검색용)
CREATE INDEX idx_pets_location ON missing_pets USING GIST(location_point);
CREATE INDEX idx_pets_district ON missing_pets(district);
CREATE INDEX idx_pets_status   ON missing_pets(status);
```

### 4.2 pgvector — 임베딩 테이블

```sql
-- pgvector 확장 활성화
CREATE EXTENSION IF NOT EXISTS vector;

-- 반려동물 특징 임베딩
CREATE TABLE pet_embeddings (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pet_id          UUID REFERENCES missing_pets(id) ON DELETE CASCADE,
    embedding       VECTOR(1536),              -- 임베딩 차원 수 (모델에 따라 조정)
    feature_text    TEXT,                      -- Claude가 생성한 특징 문장 (디버깅용)
    created_at      TIMESTAMPTZ DEFAULT now()
);

-- HNSW 인덱스 (고성능 ANN 검색)
CREATE INDEX idx_pet_embeddings_hnsw
ON pet_embeddings
USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);
```

## 5. Two-Stage Filtering 성능 분석

### 5.1 필터링 단계별 후보군 축소 효과

```
전체 실종 건 DB (예: 10,000건)
        │
        │  Stage 1: 품종 필터
        │  (동일 품종만 통과)
        ▼
품종 일치 후보군 (예: ~500건, 95% 제거)
        │
        │  위치 필터 (옵션, 반경 N km)
        ▼
위치 + 품종 후보군 (예: ~50건)
        │
        │  Stage 2: 벡터 유사도 검색
        ▼
상위 3건 최종 결과
```

### 5.2 정확도 vs. 재현율 트레이드오프

| 전략 | 장점 | 단점 |
|---|---|---|
| 품종 필터 Strict | 검색 속도 향상, 노이즈 감소 | 품종 오인식 시 실제 매칭 누락 위험 |
| 품종 필터 Soft (신뢰도 임계값) | 오인식 케이스 포괄 | 후보군 증가 → 검색 비용 증가 |
| 위치 필터 추가 | 지역 맥락 반영, 정밀도 향상 | GPS 정보 없는 케이스 처리 필요 |

**권장 전략:**
- 품종 신뢰도 ≥ 0.7 → 품종 필터 적용
- 품종 신뢰도 < 0.7 → 필터 스킵, 전체 후보 대상 벡터 검색
- 위치 정보 존재 시 → 반경 10km 필터 추가 적용

## 6. 비동기 처리 설계

요구사항 명세서 §3.3에 따라 AI 분석은 수 초 이상 소요되므로 **비동기 처리**가 필수입니다.

```
POST /api/tips/analyze
        │
        ▼
즉시 응답: { tipId: "tip_xxx", status: "processing" }
        │
        ▼ (백그라운드 큐)
[Stage 1] The Cat API 호출
[Stage 2] Claude Sonnet 특징 증강
[Stage 3] 임베딩 생성
[Stage 4] pgvector 유사도 검색
        │
        ▼
DB에 결과 저장 (tips 테이블)
        │
        ├── 클라이언트 폴링: GET /api/tips/:tipId
        │   { status: "done", results: [...] }
        │
        └── WebSocket 푸시: { event: "tip.complete", tipId, results: [...] }
```

### 진행률 단계 매핑

| 단계 | 진행률 | 메시지 |
|---|---|---|
| 요청 접수 | 0% | 처리 시작 |
| The Cat API 완료 | 25% | "데이터베이스 조회 중" |
| Claude Sonnet 완료 | 60% | "특징점 추출 중" |
| 임베딩 + 검색 완료 | 100% | "유사도 매칭 중" |

## 8. 기술 스택 요약

| 레이어 | 기술 | 비고 |
|---|---|---|
| API Gateway | Kong / AWS API Gateway / Nginx | JWT 인증, Rate Limit |
| 서비스 레이어 | Node.js (Express/Fastify) 또는 Python (FastAPI) | 비즈니스 로직 |
| 관계형 DB | PostgreSQL + PostGIS | 지오공간 쿼리 지원 |
| 벡터 DB | pgvector (PostgreSQL 확장) | 별도 벡터 DB 불필요 |
| 품종 분류 | The Cat API | Stage 1 필터 |
| LLM 특징 증강 | Claude Sonnet (Anthropic API) | Stage 2, 멀티모달 |
| 임베딩 | text-embedding-3-small 또는 multilingual 모델 | 한국어 지원 필수 |
| 실시간 | WebSocket (ws / Socket.io) | 채팅, 알림 |
| 푸시 알림 | FCM (Android) + APNs (iOS) | 반경 2km 발송 |
| 파일 스토리지 | AWS S3 / GCS | Presigned URL 방식 |
| 큐 | Redis (BullMQ) / AWS SQS | AI 파이프라인 비동기 처리 |
| 캐시 | Redis | 조회수, 세션, 좋아요 캐시 |
