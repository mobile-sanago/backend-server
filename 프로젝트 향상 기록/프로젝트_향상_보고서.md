# Missing Pet Finder 프로젝트 향상 보고서

## 1. 보고서 개요
- 작성일: 2026-05-26
- 목적: 초기 스캐폴딩 상태에서 현재까지의 개선 사항을 구조적으로 정리하고, 성과/한계/후속 과제를 명확히 기록
- 범위: `server`, `ai_server`, `supabase`, `implementation_specs` 기반 백엔드 및 AI 품질 검증 체계

## 2. 초기 상태와 문제 정의
초기 상태는 서버 기동 및 헬스체크 중심이었고, 실제 서비스 운영 관점에서는 다음 공백이 있었다.
- 도메인 API 미완성(인증/게시글/채팅/제보/알림 흐름 단절)
- 운영 자동화 부족(프로세스 관리, 큐/캐시 처리, 장애 시 확인 루프)
- AI 품질을 수치화할 검증 루프 부재(정확도/지연/실패율 추적 불가)

## 3. 향상 내용

### 3.1 도메인 기능 구현 확장
- 인증/사용자, Pets CRUD, Like/Comment, Chats, Tips, Notifications, Uploads, Devices API 경로를 구현/보강했다.
- 실사용 테스트 경로를 위해 Swagger 노출을 확장했다.
  - `/docs`
  - `/docs/openapi.json`

### 3.2 운영 가능 구조로 전환
- PM2 기반 다중 프로세스(`server`, `ai_server`, `ai_worker`) 운영 루프를 정비했다.
- Supabase/Redis/BullMQ/FCM 연계를 구성해, 단순 기동이 아닌 비동기 처리/알림 흐름을 운영 가능한 형태로 만들었다.
- 재시작/로그/상태 확인이 가능한 운영 점검 루프를 확보했다.

### 3.3 AI 품질 검증 체계 구축
- 30건 검증 루프를 먼저 만든 뒤, 100건으로 확장했다.
- 배치 평가/오분류 분석/리포트 생성을 자동화해 성능 개선을 반복 가능한 구조로 전환했다.
- 외부 provider 상태를 사전 점검하는 게이트를 추가했다.
  - `ai_server/scripts/check_ai_provider_status.py`

### 3.4 실사진 평가 루프 신설
- `CAT_00` 실사진(1,308장)을 기반으로 실데이터 검증 경로를 만들었다.
- 무라벨 100셋 생성 -> 예측 -> runmeta -> 리포트의 일관 파이프라인을 추가했다.
- 실패 샘플 재처리 안정성을 높이기 위해 재시도/부분 재실행 기능을 추가했다.
  - `--max-retries`
  - `--retry-sleep-ms`
  - `--only-errors-from`
  - `merge_prediction_results.py`

### 3.5 최신: Claude 준라벨 GT v1.5 도입
- 수동 GT 이전 단계로 운영 비교 가능한 기준선을 만들기 위해 준라벨 생성기를 추가했다.
  - `generate_real_gt_v1_5_with_claude.py`
- 품질 재산출 스크립트에 외부 GT 재사용과 버전 기록 옵션을 추가했다.
  - `--use-existing-ground-truth`
  - `--ground-truth-version`
  - `--skip-prediction`
- 주의: v1.5는 비수동 GT이므로 정식 성능 판정 기준이 아니다.

## 4. 성과 지표

### 4.1 합성/정답셋(100) 기준
- `success=100/100`
- `top1_accuracy=1.00`
- `latency_p95_ms=10146`

### 4.2 실사진/사람검수 GT(v2-human) 기준
- `top1_accuracy=0.5667`
- `pred_unknown_rate=0.51`
- `success_rate=1.0`
- `latency_p95_ms=2915596`

해석:
- 합성셋에서 정확도/지연 목표는 달성했다.
- 실사진에서는 실패 건은 복구됐지만 지연 이상치가 커, 지표 분해와 재검증이 필요하다.

### 4.3 정확도 향상 추이
기록된 배치 결과 기준으로 정확도는 다음 흐름으로 개선되었다.
- 30건 배치 초기: `top1_accuracy=0.80`
- 100건 확장 1차: `top1_accuracy=0.84`
- 외부 한도 이슈 복구 직후 변동 구간: `0.82 -> 0.83`
- 보정 2차(합성/정답셋) 이후: `top1_accuracy=1.00` 달성
- 실사진/준라벨(v1.5-auto-claude) 기준선: `top1_accuracy=0.5667`

해석:
- 합성/정답셋에서는 프롬프트/분류 보정 효과로 정확도가 단계적으로 개선되어 목표를 초과 달성했다.
- 실사진은 데이터 난이도와 라벨 신뢰도(비수동 GT) 영향으로 정확도가 낮게 측정되며, 사람 검수 GT(v2) 기반 재평가가 필요하다.

## 5. 남은 리스크 및 개선 과제
- 실사진 실패 3건은 복구 완료(현재 100/100 성공), 다만 provider gate가 날짜별로 변동(2026-05-26 점검 실패)하여 재예측 안정성 리스크 존재
- 실사진 p95 지연 이상치로 운영 지표 해석 왜곡 가능
- `claude-sonnet-4-20250514` 모델 EOL(2026-06-15) 대응 필요(후보 비교 완료, 즉시 교체는 보류)

우선순위:
1. provider gate 정상화 후 실사진 100 재예측 재실행
2. 실사진 지연 지표 분해(정상 응답 p95 / 전체 p95 / outlier Top N)
3. Claude 후보 모델 동일 조건 회귀 비교 후 채택/롤백 확정
4. 최종 리포트/운영 문서 수치 동기화

## 6. 결론
프로젝트는 “기동 확인 중심”에서 “운영 및 품질 개선이 가능한 체계”로 의미 있게 향상되었다.  
현재 핵심 과제는 실사진 기준 신뢰도 확보이며, 이를 위해 사람 검수 GT(v2)와 모델 교체 대응이 다음 단계의 결정적 작업이다.
